# Quick Start

Use this path when adopting the template into a new repository.

1. Create or prepare the target repository.
2. Adopt the template governance surfaces.
3. Read `AGENTS.md`, `.codex/AGENTS.md`, and `docs/bootstrap/governed-project-bootstrap.md`.
4. Run `docs/pipelines/governance/000--initialize-governed-project.md` if the repository is not already governance-capable.
5. Run `docs/pipelines/governance/001--discover-existing-architecture-and-establish-doctrine.md`.
6. Run `docs/pipelines/governance/002--audit-repository-governance-readiness.md`.
7. Use pipelines `003` through `007` only when the repository has bounded surfaces that need contract governance, audits, remediation, or verification.

Most repositories can reach a valid initial governed state after steps 1 through 6.
