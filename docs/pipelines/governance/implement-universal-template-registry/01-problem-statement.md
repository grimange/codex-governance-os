# Problem Statement

Before this pipeline, `codex-governance-os` had template-family definitions but no admitted registry of concrete template identities. That left several unresolved risks:

- template bodies were not admitted as first-class governed assets
- no compiled registry index existed
- no deterministic template resolution contract existed
- lifecycle states such as restricted, deprecated, and archived were not enforced for template identities
- registry evidence commands were not available for audit and verification work

This pipeline closes that gap by introducing a governed registry model distinct from the family template registry.
