# Repository Entrypoint Design

## README Structure

- repository purpose
- start-here links
- what the template provides
- adoption profiles
- scope and neutrality statement
- deeper links into adoption, lifecycle, and skill discovery

## Quick-Start Entry

The entrypoint routes a new user first to:

1. `docs/adoption/quick-start.md`
2. `docs/adoption/codex-governance-adoption-guide.md`
3. `docs/bootstrap/governed-project-bootstrap.md`

## Required Overviews

- governance lifecycle overview linked through `docs/governance/governance-lifecycle.md`
- universal skills overview linked through `docs/governance/universal-skills-index.md`
- active pipeline catalog linked through the registry

The entrypoint is designed so a new user can understand the framework, the minimum path, and the deeper references within a few minutes.
