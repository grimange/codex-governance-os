# Normalized Gap Analysis Structure

Pipeline 155 establishes the following canonical top-level section model for
`docs/governance/governance-maturity-gap-analysis.md`:

1. Purpose
2. Capability Model
3. Coverage Assessment Method
4. Evidence Boundaries
5. Interpretation Boundaries
6. Update Discipline
7. Current Scorecard
8. Capability Gaps and Blockers
9. Next Advancement Targets

## Section Intent

- `Purpose`
  - defines the role of the surface
- `Capability Model`
  - lists the stable diagnostic capability areas
- `Coverage Assessment Method`
  - defines normalized coverage states and their meaning
- `Evidence Boundaries`
  - states which repository truth sources are admissible
- `Interpretation Boundaries`
  - constrains how the diagnostic surface may be read
- `Update Discipline`
  - defines safe maintenance and refresh behavior
- `Current Scorecard`
  - records the current bounded maturity context that the engine explains
- `Capability Gaps and Blockers`
  - records coverage status and material maturity blockers
- `Next Advancement Targets`
  - separates forward-looking diagnostic targets from current-state claims
