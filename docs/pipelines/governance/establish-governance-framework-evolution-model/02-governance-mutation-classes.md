# Governance Mutation Classes

| class | meaning | examples | compatibility expectation |
|-------|---------|----------|---------------------------|
| editorial clarification | wording or discoverability changes that do not alter effective governance behavior | typo fixes, clearer cross-links, concise wording improvements | compatible |
| additive compatible extension | new optional governance surface or rule that does not force adopters to restructure | new optional adoption guide, new optional pipeline | compatible |
| behavioral refinement | bounded change to governance interpretation or workflow behavior | clarifying when a pipeline is conditional, tightening artifact expectations | assess explicitly |
| breaking governance change | change that alters active governance meaning or required structure materially | authority-order change, required-surface removal, incompatible pipeline repurposing | incompatible unless migrated |
| deprecation-managed transition | replacement of an existing governance surface through a compatibility window | superseding a guide or doctrine surface with a new canonical one | transitional |
| emergency corrective repair | urgent fix restoring intended governance safety | registry identity repair, doctrine correction that removes dangerous ambiguity | usually compatible if scope is bounded |

These classes govern how future governance mutations are planned, reviewed, and promoted.
