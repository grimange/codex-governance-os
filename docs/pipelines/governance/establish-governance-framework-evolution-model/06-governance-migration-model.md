# Governance Migration Model

## Migration Triggers

Migration guidance is required when a governance mutation:

- changes required repository structure
- changes authority interpretation
- replaces a canonical doctrine or adoption path
- changes active pipeline usage materially
- changes how inherited and local governance surfaces interact

## Migration Requirements

- state the old model and new model clearly
- enumerate required update steps
- identify any compatibility period or deprecation window
- record the migration in pipeline artifacts so later audits can trace the transition

## Preferred Execution Pattern

Use an explicit governance pipeline when a pipeline exists. If no suitable pipeline exists, record why a narrower bounded path was safe.
