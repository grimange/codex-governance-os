# Governance Deprecation Model

## Deprecation Rules

- Deprecate before removal when a governance surface is already published or adopted broadly, unless an emergency corrective repair exception is justified.
- Mark the preferred replacement in doctrine or pipeline artifacts.
- Keep deprecated surfaces inspectable until the recorded transition condition is satisfied.

## Required Deprecation Record

A deprecation record should state:

- deprecated surface
- replacement surface
- reason for deprecation
- compatibility expectations during transition
- sunset or review condition

## Current State

No active doctrine or pipeline surface is being deprecated by pipeline `015`. This phase establishes the model for future use.
