# Governance Surface Inventory

| surface class | current locations | role |
|---------------|-------------------|------|
| constitutional rules | `AGENTS.md`, `.codex/AGENTS.md` | define authority ordering and local governance operating behavior |
| doctrine foundation | `docs/governance/` | define lifecycle, terminology, artifact, naming, skill, and architecture law |
| bootstrap and adoption surfaces | `docs/bootstrap/`, `docs/adoption/`, `README.md` | define the governed adoption path and public framework presentation |
| active pipeline definitions | `docs/pipelines/governance/` | define canonical governance workflows |
| registry surface | `docs/pipelines/registry/pipeline-registry.md` | define active pipeline discoverability |
| execution evidence | `docs/pipelines/governance/<pipeline-name>/` | record governance decisions, verification, and promotions |

No separate mutation queue model currently exists as an independent surface. Meta-governance control is therefore best established as doctrine plus documented pipeline execution evidence.
