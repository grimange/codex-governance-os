# Case Model Definition

Pipeline `126` establishes the session reconstruction case as one bounded
evaluation package for a reconstructed governed session.

Every case must declare:

- `session_id`
- reconstruction scope
- verification objective
- evaluation timestamp
- evaluator context
- explicit evidence sources
- explicit assumptions, inference boundaries, and ambiguity markers when
  needed

The case model is input structure for the harness, not an alternative
evaluation engine.
