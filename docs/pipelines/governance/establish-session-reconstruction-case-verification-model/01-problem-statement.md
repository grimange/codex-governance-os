# Problem Statement

Pipelines `124` and `125` established and verified the session reconstruction
verification harness.

What remained implicit was the canonical structure of an individual
reconstruction case presented to that harness.

Pipeline `126` closes that gap by establishing one doctrine-level case model
that:

- anchors every case on canonical `session_id`
- requires explicit reconstruction scope and evidence declaration
- preserves assumptions and ambiguity as explicit restrictions
- exposes consistent evaluation dimensions across reconstruction cases
- keeps case outcomes bounded by the existing harness doctrine
