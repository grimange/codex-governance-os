# Evidence-Scoped Planning Boundary

The roadmap remains bounded by repository truth.

Current bounded behavior:

- `multi_agent_governance` remains the recommended next target because it is the
  first remediation entry and the only `INVALID_STATE` blocker
- `architecture_advisor` and `autonomous_governance_loop` remain later-stage
  targets because they are `UNVERIFIED`, not invalid
- the roadmap does not invent concrete pipeline recommendations beyond what the
  remediation planner already supports

If remediation uncertainty is present upstream, the roadmap preserves that
uncertainty rather than normalizing it.
