# Governance Maturity Domain Model

Pipeline 164 defines these governance maturity domains:

- `codex_governance_layer`
- `pipeline_governance`
- `passive_observation`
- `autonomous_governance_loop`
- `multi_agent_governance`
- `architecture_advisor`

Current capability-to-domain mapping:

- `codex_governance_layer`
  - Governance Doctrine
  - Execution Governance
- `pipeline_governance`
  - Pipeline Governance
- `passive_observation`
  - Observability
- `autonomous_governance_loop`
  - Autonomous Governance Loop
- `multi_agent_governance`
  - Multi-Agent Governance
- `architecture_advisor`
  - Architecture Advisor
