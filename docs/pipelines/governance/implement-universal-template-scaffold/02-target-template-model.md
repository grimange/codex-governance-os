# Target Template Model

The implemented scaffold follows four layers:

1. Governance core
2. Universal template surface
3. Specialization overlay surface
4. Project realization surface

## Required Surfaces

- `docs/governance/`
- `docs/pipelines/`
- `docs/pipelines/governance/`
- `docs/codex/templates/`
- `docs/governance/registries/templates/`
- `tools/governance/`
- `tools/templates/`
- `artifacts/`

## Optional Runtime Surfaces

- `config/`
- `scripts/`
- `src/`
- `app/`
- `backend/`

## Overlay Rule

Overlays add runtime shape only. They must not redefine the governance core.
