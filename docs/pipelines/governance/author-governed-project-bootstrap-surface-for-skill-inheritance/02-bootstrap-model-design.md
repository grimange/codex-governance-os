# Bootstrap Model Design

## Universal Assets Inherited

- doctrine foundation under `docs/governance/`
- universal skill library and skill invocation rules
- normalized governance pipeline definitions
- pipeline registry model and artifact conventions

## Assets Created Locally By A Future Repository

- local `AGENTS.md`
- local `.codex/AGENTS.md`
- repository-specific architecture doctrine after discovery
- local contracts, local pipelines, and local skill extensions when justified

## Inheritance Model

- Doctrine inheritance works by adopting the generic governance foundation, then supplementing it with project-specific doctrine discovered locally.
- Universal skill inheritance works by using the template's reusable skills as the default operational layer for generic governance workflows.
- Local override works through higher-authority local repository files and bounded local skills under `.codex/skills/`.
- Local pipelines may coexist with inherited skills by referencing the inherited skill library while keeping local artifact and promotion rules explicit.

## Separation Rules

- Universal template law stays generic and reusable.
- Local repository constitution names project mission and routing.
- Local contracts define repository-specific subsystem expectations only after discovery.
- Local pipelines govern recurring repository-specific procedures.
- Local skills extend or narrow inherited workflows without replacing the template baseline casually.

## Post-Bootstrap Architecture Establishment

Bootstrap does not attempt to define a future repository's architecture. After initialization, the next authoritative step is local architecture discovery and doctrine authoring.
