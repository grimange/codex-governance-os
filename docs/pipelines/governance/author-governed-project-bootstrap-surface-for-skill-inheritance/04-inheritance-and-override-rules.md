# Inheritance And Override Rules

## Default Inheritance

- Future repositories inherit the doctrine foundation under `docs/governance/`.
- Future repositories inherit the universal skills foundation as the default reusable behavior layer.
- Future repositories inherit the normalized governance pipeline model and registry discipline.

## Local Override Conditions

- Local repository rules may narrow inherited guidance when repository-specific needs exist.
- Local project skills may override a universal skill only when the override is explicit, discoverable, and consistent with higher-authority doctrine.
- Local doctrine may supplement inherited doctrine, but it should not silently erase core authority, artifact, or verification rules.

## Conflict Resolution

1. version-controlled repository state
2. local repository constitution
3. local architecture doctrine plus inherited doctrine foundation
4. active pipeline definitions
5. generated pipeline artifacts

## Extension Discipline

- Local skills extend rather than pollute the universal library by living under `.codex/skills/`.
- Local contracts are introduced only after local discovery establishes stable subsystem boundaries.
- Project-specific architecture doctrine is established after bootstrap through explicit discovery and doctrine authoring work.

## Stability Rule

The inherited universal foundation is the baseline. Local specialization is allowed, but it must remain bounded, inspectable, and explicitly justified in repository state.
