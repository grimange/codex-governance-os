# Harness Model Definition

Pipeline `124` establishes the following verification-harness model:

1. take a claimed reconstructed session narrative as input
2. anchor that narrative on canonical `session_id`
3. verify that cited lifecycle claims are traceable to admissible canonical
   evidence
4. evaluate lifecycle meaning through the state-machine canon and evidence
   precedence through the interpretation model
5. confirm that lifecycle observation and runtime context remain subordinate
6. verify that the evidence set supports one bounded deterministic narrative
7. classify the result as `VERIFIED`, `VERIFIED_WITH_RESTRICTIONS`, or
   `FAILED`

This model operationalizes verification of reconstruction outputs without
collapsing the reconstruction procedure and the verification procedure into one
surface.
