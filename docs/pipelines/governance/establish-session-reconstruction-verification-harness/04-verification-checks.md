# Verification Checks

The harness doctrine requires the following checks:

- evidence anchoring to canonical `session_id`
- ledger consistency so reconstructed events are not invented
- state-machine conformance for all claimed lifecycle movement
- evidence-precedence compliance under the session interpretation model
- observation subordination so normalized lifecycle signals do not outrank
  ledger evidence
- runtime-boundary compliance so runtime context remains supporting only
- deterministic narrative verification so one bounded valid narrative exists

Failure of any required check causes verification failure rather than silent
normalization.
