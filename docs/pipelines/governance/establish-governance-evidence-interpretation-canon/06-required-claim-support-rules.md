# Required Claim Support Rules

The canon defines minimum evidence expectations for the main governance claim
classes used throughout the repository:

- establishment claims
  - require created or updated governing surfaces plus explicit scope
- verification claims
  - require executed procedures and recorded results tied to the scoped claim
- restriction claims
  - require explicit boundary recording and evidence that the repository does
    not silently treat the boundary as supported
- blocked claims
  - require the missing or conflicting prerequisite to be stated explicitly
- stability or non-drift claims
  - require re-verification against governing surfaces after prior establishment

This closes the Layer 0 gap where verdict words existed in practice but their
support requirements were not centralized as one doctrine.
