# Problem Statement

The repository already has:

- a centralized governance memory in `docs/governance/pipeline-run-ledger.md`
- a derived analytics surface in `docs/governance/pipeline-run-analytics.md`
- a deterministic verdict taxonomy in `docs/governance/verdict-family-classification-canon.md`

What it does not yet have is a canonical scorecard that interprets those
surfaces into a bounded maturity reading.

Without a maturity scoring surface, governance progress remains visible but not
systematically evaluated. Pipeline 149 closes that gap by creating a derived
reporting document that converts recorded governance evidence into explicit
maturity dimensions and scores.
