# Cross-Role Interaction Verification

The role model defines the required interaction discipline:

- planner classifies and routes
- integrator maintains canonical surfaces
- verifier validates results
- restriction keeper preserves fail-closed boundaries
- architecture auditor guards doctrine integrity
- evidence reporter records bounded outcomes

Safe escalation behavior is also explicit:

1. fail closed
2. report the boundary
3. recommend the next safe lane or governed path

Restriction preserved by this verification:

- interaction is defined as doctrine only
- runtime orchestration or automatic delegation is not claimed
