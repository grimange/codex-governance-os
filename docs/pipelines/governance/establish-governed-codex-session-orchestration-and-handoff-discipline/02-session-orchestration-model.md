# Session Orchestration Model

The canonical Layer 6 doctrine is established at
[layer-6-codex-session-orchestration-and-handoff-discipline.md](/home/ramjf/python-projects/codex-governance-os/docs/governance/layer-6-codex-session-orchestration-and-handoff-discipline.md).

Established model:

- a bounded hub-and-spoke orchestration discipline
- one orchestrator session per governed workstream
- scoped subordinate sessions or sub-agents
- no authority gain through orchestration language alone
- no uncontrolled concurrent structural mutation

The model is intentionally doctrine-only. It governs how orchestration should
be reasoned about in repository governance work, not how runtime infrastructure
is implemented.
