# Prohibited Behaviors

Layer 6 explicitly prohibits:

- silent task reassignment
- hidden session forks affecting governed work
- uncontrolled concurrent structural edits
- undocumented handoffs
- skipping required verification
- broadening authority through orchestration language alone

The doctrine also preserves these boundary conditions:

- session coordination does not prove runtime orchestration support
- session coordination does not authorize automatic delegation
- session coordination does not create authority beyond Layers 0 through 5

These prohibitions keep session orchestration subordinate to the lower-layer
governance canons instead of treating coordination as a new source of power.
