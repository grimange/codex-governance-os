# Layer 3 Codex Rules Canon

The canonical Layer 3 surface established by this lane is:

- [layer-3-codex-rules-canon.md](/home/ramjf/python-projects/codex-governance-os/docs/governance/layer-3-codex-rules-canon.md)

The canon defines:

- the authority order Codex must respect
- the minimum request classes Codex must distinguish
- when governed routing is required
- how edits must remain bounded and restriction-preserving
- how claims and verdicts must remain evidence-scoped
- how Codex must behave during verification and remediation
- how unsupported, ambiguous, or stale situations must fail closed
- how status reporting and recommendations must stay separated

This first Layer 3 canon is intentionally bounded and single-agent oriented.
