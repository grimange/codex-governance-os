# Session Registry Model

The canonical session registry is established at
[codex-session-registry.md](/home/ramjf/python-projects/codex-governance-os/docs/governance/codex-session-registry.md).

Established model elements:

- deterministic `CS-YYYYMMDD-###` session identifiers
- supported session-type taxonomy
- orchestrator and parent-session relationship fields
- lifecycle status recording
- final verdict and primary-scope recording

The registry begins as an empty canonical table. This lane establishes the
recording model, not retrospective historical entries for prior sessions.
