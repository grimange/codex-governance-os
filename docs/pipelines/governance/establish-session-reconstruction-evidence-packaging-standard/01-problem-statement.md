# Problem Statement

Pipelines `124` through `127` established and verified the reconstruction
verification harness and the reconstruction case model.

What remained implicit was how evidence used by those cases must be organized
and preserved.

Pipeline `128` closes that gap by establishing one doctrine-level packaging
standard that:

- anchors each package on canonical `session_id`
- requires one consistent set of evidence sections
- preserves deterministic evidence ordering
- keeps citations, assumptions, and restrictions explicit
- remains compatible with the existing harness and case model
