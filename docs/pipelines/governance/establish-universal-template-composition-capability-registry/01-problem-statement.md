# Problem Statement

Pipelines `045` through `049` established a certified and continuously verified template composition matrix, but the composition model still expressed most of its truth through explicit overlay-pair admission.

That works for the current overlay set, but it scales poorly as future overlays introduce reusable concerns such as worker runtimes, topology orchestration, service hosts, and framework boundaries.

Pipeline `050` establishes a governed capability vocabulary so template manifests can declare composition intent in a reusable form while preserving the already-certified matrix as the active authority surface.
