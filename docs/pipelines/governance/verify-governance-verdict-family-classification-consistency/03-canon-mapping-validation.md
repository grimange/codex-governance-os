# Canon Mapping Validation

Canonical mapping rules from `docs/governance/verdict-family-classification-canon.md` were applied in first-match order.

- `_WITH_GAPS` -> `WITH_GAPS`
- `_WITH_(LIMITATIONS|LIMITATION|RESTRICTION|RESTRICTIONS)` -> `WITH_LIMITATIONS`
- `NORMALIZED` -> `NORMALIZED`
- `VERIFIED` -> `VERIFIED`
- `ESTABLISHED` -> `ESTABLISHED`
- `_IMPLEMENTED` -> `IMPLEMENTED`
- `_RESTRICTED` -> `RESTRICTED`
- fallback -> `OTHER`

All `60` ledger verdicts mapped to exactly one family under that order, including the fallback `OTHER`.
