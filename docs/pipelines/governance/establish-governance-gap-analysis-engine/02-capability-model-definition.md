# Capability Model Definition

Pipeline 153 defines the following normalized governance capability areas:

- Governance Doctrine
- Pipeline Governance
- Execution Governance
- Observability
- Governance Intelligence
- Multi-Agent Governance
- Autonomous Governance
- Architecture Advisory

These capability areas are cross-cutting coverage categories. They are not a
replacement for the repository's architectural layers.

Allowed coverage states:

- `fully implemented`
- `partially implemented`
- `emerging capability`
- `not implemented`
- `intentionally out of scope`
