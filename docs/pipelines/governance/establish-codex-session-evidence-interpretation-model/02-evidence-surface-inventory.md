# Evidence Surface Inventory

The new interpretation model is grounded in the following canonical surfaces:

- `docs/contracts/codex-session-state-machine-canon.md`
  - authoritative lifecycle meaning
- `docs/governance/codex-session-registry.md`
  - canonical session identity and indexed summary state
- `docs/governance/codex-session-ledger.md`
  - durable event evidence
- `docs/governance/codex-session-lifecycle-observation-discipline.md`
  - normalization of lifecycle signals into canonical evidence
- `docs/governance/codex-session-runtime-boundary-and-evidence-model.md`
  - admissible runtime-supporting context and mapping constraints
- `docs/governance/governance-evidence-interpretation-canon.md`
  - repository-wide evidence sufficiency and fail-closed interpretation rules

Pipeline `118` adds a session-specific interpretation layer that applies these
surfaces together without replacing any of them.
