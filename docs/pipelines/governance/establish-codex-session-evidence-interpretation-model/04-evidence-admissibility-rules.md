# Evidence Admissibility Rules

The session-evidence interpretation model treats evidence as admissible when
it is:

- produced by a canonical Layer 6 governance surface
- recorded in the session ledger
- recorded in the session registry
- derived through lifecycle-observation normalization from canonical evidence
- admitted as supporting runtime context through the runtime-boundary canon

The model rejects or bounds evidence when it:

- cannot be mapped back to `session_id`
- implies lifecycle semantics not defined by the state machine
- exists only as unsupported narrative summary
- originates outside the canonical session-governance surfaces without an
  admissible mapping path

Interpretation remains fail-closed when evidence is partial or conflicting.
