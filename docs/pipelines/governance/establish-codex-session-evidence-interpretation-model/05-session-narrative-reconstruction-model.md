# Session Narrative Reconstruction Model

Pipeline `118` establishes the following reconstruction sequence for governed
session truth:

1. identify the session through `session_id` in the registry
2. gather applicable ledger entries for that session
3. interpret lifecycle implications through the state-machine canon
4. use lifecycle-observation normalization where clarification is needed
5. consult runtime-boundary evidence only as supporting context that maps back
   to canonical fields
6. preserve remaining ambiguity, insufficiency, or restriction explicitly

This model produces governance-observable session truth rather than claiming
complete runtime capture.
