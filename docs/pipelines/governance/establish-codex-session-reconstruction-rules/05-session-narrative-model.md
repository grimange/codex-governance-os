# Session Narrative Model

The reconstruction rules define the output as one bounded
governance-observable session narrative.

That narrative:

- begins from canonical `session_id`
- reflects durable ledger evidence rather than inferred event invention
- uses state-machine meaning for lifecycle interpretation
- incorporates observation normalization only where admissible
- uses runtime context only as supporting evidence
- preserves ambiguity and insufficiency explicitly when the evidence does not
  support a stronger claim

This output is a governed reconstruction, not a claim of full runtime capture.
