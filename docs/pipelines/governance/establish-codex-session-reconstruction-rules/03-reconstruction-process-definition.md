# Reconstruction Process Definition

Pipeline `121` establishes the following deterministic reconstruction process:

1. identify the governed session through canonical `session_id`
2. collect applicable ledger evidence for that session
3. normalize admissible lifecycle signals through the observation doctrine
4. interpret lifecycle meaning through the state-machine canon
5. admit runtime context only when it maps back into canonical fields
6. construct one bounded governance-observable narrative that preserves any
   remaining ambiguity, insufficiency, or restriction explicitly

This process operationalizes the interpretation model without creating new
authority surfaces.
